# Ideation Frameworks

Use these selectively. Do not run every framework mechanically.

## How Might We

Reframe a vague idea into an opportunity:

- "How might we <desired outcome> for <specific user> without <key constraint>?"
- Good framing is narrow enough to act on and broad enough to allow multiple solutions.
- Avoid solution-embedded prompts such as "How might we add a chatbot?"

## SCAMPER

Useful for improving or reimagining an existing product, feature, or workflow.

- Substitute: replace audience, technology, business model, or process.
- Combine: merge with an adjacent idea or capability.
- Adapt: borrow from another industry, domain, or time period.
- Modify: magnify, minimize, exaggerate, or strip down.
- Put to other uses: find a new user or problem for the same capability.
- Eliminate: remove steps, configuration, roles, or features.
- Reverse or rearrange: invert the order, ownership, or value chain.

## First Principles

Use when the idea is anchored to convention.

1. List what is actually true.
2. List what is assumed.
3. Challenge assumptions that are not real constraints.
4. Rebuild from the truths only.

## Jobs To Be Done

Use when the user or problem is unclear.

- Functional job: what task is the user trying to complete?
- Emotional job: how do they want to feel?
- Social job: how do they want to be perceived?
- Format: "When I <situation>, I want to <motivation>, so I can <outcome>."

## Constraint-Based Ideation

Use constraints to force focus:

- one day to build
- one feature only
- no obvious technology
- free forever
- non-expert user
- 10 users vs. 1 billion users

## Pre-Mortem

Use during evaluation:

1. Imagine the idea shipped and failed.
2. List technical, market, timing, adoption, and team failure modes.
3. Decide which are preventable, acceptable, or fatal.

## Analogous Inspiration

Look for structural parallels, not superficial "X for Y" analogies.

- What other domain solved a similar trust, coordination, discovery, or workflow problem?
- What would this look like if built for a different audience or environment?
- What pattern from another industry transfers cleanly?
