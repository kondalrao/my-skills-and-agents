# Layout Patterns

Use these patterns as starting points. Choose the one that best matches the user goal instead of forcing every poster into the same template.

## Section Bands

Best for use-case posters, reference sheets, and operational summaries.

- Header with one decision sentence.
- 4-7 horizontal bands.
- Each band has a color rail, section title, 3-5 concrete bullets, tags, and a "good when" or tradeoff callout.

## Bento Grid

Best for mixed importance content.

- Large primary card for the main concept.
- Smaller cards for sections, examples, metrics, and caveats.
- Use different card sizes to signal priority.

## Radial Hub

Best for ecosystem maps and platform concepts.

- Central concept in the middle.
- 4-7 orbiting sections.
- Connectors indicate relationships.
- Add side callouts for tradeoffs and implementation rules.

## Matrix

Best for comparisons and decision support.

- Rows are use cases, tools, controls, or phases.
- Columns are value, attach point, maturity, risk, and example.
- Use compact cells, badges, and color-coded fit indicators.

## Architecture Map

Best for systems, network, security, and runtime-control posters.

- Draw layers from user/app to kernel/runtime to network/storage.
- Place hooks, probes, controls, and data flows on the relevant layer.
- Keep labels close to edges and flow arrows.

## Timeline Or Runbook

Best for incident, debugging, or rollout posters.

- Left-to-right or top-to-bottom phases.
- Each phase has trigger, action, signal, and failure mode.
- Include a compact checklist or command strip.
