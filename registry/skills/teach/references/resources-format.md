# Resources Format

`RESOURCES.md` is the curated source list for one topic directory. Use it to ground lessons in trusted knowledge and to identify communities where the user can get real-world feedback.

## Template

```markdown
# <Topic> Resources

## Knowledge

- [<Resource title>](<url>) - <What it covers and when to use it.>

## Wisdom

- [<Community, class, forum, mentor, or practice setting>](<url>) - <What feedback it can provide and when it is useful.>

## Gaps

- <Missing source or open question that should be researched before teaching this area deeply.>
```

## Rules

- Prefer official docs, primary sources, recognized experts, peer-reviewed work, and well-moderated communities.
- Annotate every entry. A bare link is not useful later.
- Group sources as `Knowledge` for information and `Wisdom` for practitioner feedback.
- Prune weak, stale, off-mission, or marketing-heavy sources.
- Record user preferences such as "do not recommend communities" so future sessions respect them.
- For current, medical, legal, financial, safety, or fast-changing technical topics, refresh sources before relying on older entries.
