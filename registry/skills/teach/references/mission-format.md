# Mission Format

`MISSION.md` lives in each topic directory under:

```text
/Users/kkomarag/Documents/ObsidianNotes/40 Knowledge/AI/Study/<topic-slug>/
```

It is the compass for that learning topic. Every lesson, resource choice, reference sheet, and practice activity should trace back to it.

## Template

```markdown
# Mission: <Topic>

## Why

<One to three sentences explaining the concrete real-world outcome the user wants. Avoid vague goals like "understand X" when the real goal is to build, debug, pass an exam, teach, ship, or decide something.>

## Success Looks Like

- <Specific observable capability>
- <Specific observable capability>
- <Specific observable capability>

## Constraints

- <Time, budget, tools, prior commitments, learning preferences, or other bounds>

## Out Of Scope

- <Adjacent topics intentionally deferred for now>
```

## Rules

- Keep one mission per topic directory.
- Make the mission concrete enough to choose the next lesson.
- Interview the user before writing a vague mission.
- Confirm with the user before changing an existing mission.
- Keep the file short. If it becomes a full plan, compress it back into a compass.
